=== PROPOSED IMPROVEMENTS ===

CATEGORY A: SAFETY FILTERS (Pre-entry checks)

A1. MINT/FREEZE AUTHORITY CHECK
Add verification that token's Mint Authority and Freeze Authority are revoked before allowing entry. Reject tokens where either is still active. Use RugCheck API or on-chain Solana account data.

A2. VOLUME vs GLOBAL FEES VALIDATION (Wash Trading Detector)
Compare token's reported volume against actual global transaction fees. If volume is high but fees are disproportionately low, flag as wash trading and reject. Rationale: fees can't be faked on-chain.

A3. LIQUIDITY LOCK VERIFICATION
Check whether LP tokens are burned or locked for a significant duration. Unlocked LP = dev can pull liquidity at any time.

A4. DEV WALLET AGE CHECK
Check the age of the token creator's wallet. Dev wallet created < 24 hours ago = red flag. Dev with history of previous rugs = auto-reject.

A5. HONEYPOT / TRANSFER HOOK DETECTION
Check for custom transfer hooks or sell restrictions that prevent holders from selling.

CATEGORY B: SIGNAL IMPROVEMENTS (Better entry decisions)

B1. SMART MONEY WALLET MONITORING AS PRIMARY SIGNAL SOURCE
We have 80+ tracked wallets of consistently profitable traders. Currently Charon only checks wallet exposure AFTER a candidate already exists (passive enrichment). Proposal: actively poll these wallets and when 3+ tracked wallets buy the same token, auto-generate a candidate.

B2. HOLDER ANALYSIS FIX — EXCLUDE LP/BURN ADDRESSES
Current holder % analysis includes LP pool addresses and burn addresses in "top holder" calculation. This makes safe tokens look risky. Fix: exclude known LP and burn addresses.

B3. 3-CANDLE DIP CONFIRMATION FOR DIP_BUY STRATEGY
Before entering on dip_buy strategy, require: (1) price touches support level, (2) volume spikes on bounce, (3) bullish candle closes above support. Use existing Jupiter chart data.

B4. DEX PAID AS CONFIDENCE SIGNAL
GMGN API returns whether a token has "DEX Paid" listing. This means dev paid $300-500 for listing — less likely to be instant rug. Use as positive signal in LLM prompt or filter score.

B5. VOLUME SPIKE DETECTION (≥200% of baseline)
Detect volume spikes ≥200% above 7-day average as accumulation signal. Currently Charon only checks absolute volume thresholds.

B6. HOLDER GROWTH VELOCITY
Track delta of unique holder count between time periods. Growing holders = organic interest. Stagnant/declining = dying token.

B7. SOCIAL MOMENTUM / TWITTER MENTION VELOCITY
Track Twitter mention frequency as a signal. Social velocity spike preceding price action = early entry opportunity.

CATEGORY C: EXIT STRATEGY IMPROVEMENTS

C1. MULTI-STAGE PARTIAL TP
Instead of single PARTIAL_TP at one threshold, implement staged exits:
- Stage 1: Sell 25% at 2x (recover initial)
- Stage 2: Sell 25% at 4x (lock profit)
- Stage 3: Trail remaining 50% with aggressive trailing stop

C2. VOLUME-BASED EXIT SIGNAL
Exit position if trading volume drops >70% from entry snapshot volume. Rationale: "volume is the heartbeat of a token — when volume dies, token dies."

C3. TRAILING STOP OPTIMIZATION
Test and tune trailing deviation between 15-25% instead of current fixed value. Consider making it adaptive based on token volatility.

C4. MCAP-TIER ADAPTIVE TP/SL
Auto-adjust TP/SL based on market cap tier:
- Micro ($10K-$100K): TP 200-500%, SL -40%
- Small ($100K-$1M): TP 50-200%, SL -25%
- Mid ($1M+): TP 30-100%, SL -15%

C5. MOMENTUM DEATH EXIT
Exit if token disappears from trending feed after being trending at entry. Logic: losing trending status = momentum death.

CATEGORY D: RISK MANAGEMENT

D1. DAILY LOSS CIRCUIT BREAKER
Auto-halt all trading after 3% portfolio daily loss OR 3 consecutive stop-loss exits.

D2. CONSECUTIVE LOSS TRACKING + PAUSE
Pause bot after N consecutive SL hits. Resume after cooldown period.

D3. DRY POWDER RESERVE
Stop opening new positions if SOL balance drops below 30% of initial balance. Always keep reserves for opportunities.

D4. ADAPTIVE POSITION SIZING (COMPOUNDING)
Instead of fixed SOL per trade: use 80% of available balance, compound winners, extract 35% of profits to cold wallet.

D5. SOL PRICE CRASH PAUSE
Pause all entries during SOL/USD crash >10%. Market-wide risk-off = all meme coins dump.

CATEGORY E: DETECTION & INTELLIGENCE

E1. CABAL PLAY DETECTION
Detect coordinated buying: multiple large wallets buying within 10-30 minute window + sudden social media push = cabal play. Use as WARNING, not entry signal.

E2. BUNDLE DETECTION PER-CANDIDATE
Beyond the existing bundler_rate trending filter: for each individual candidate, analyze holder clustering to detect if supply is bundled (many wallets funded from same source).

E3. NARRATIVE TAGGING SYSTEM
Classify tokens by narrative meta (AI, political, animal, celebrity) and adjust TP/SL per category. Political = fast scalp, Animal = moonbag potential, Celebrity = ultra-fast exit.

E4. MOMENTUM CASCADE PHASE DETECTION
Detect which phase a token is in:
- Phase 1: Smart money accumulation (on-chain visible, social quiet)
- Phase 2: Social velocity increasing
- Phase 3: Volume explosion, retail FOMO
- Phase 4: Smart money distribution → EXIT ZONE
Entry should be Phase 1-2 only. Phase 3-4 = avoid.

E5. DAY PHASE SWING STRATEGY
New strategy type: buy mid-cap tokens on weekend dips, sell on weekday volume spikes. Hold 3-7 days. Very different from current intraday approach.

E6. NETWORK GAS HEALTH INDICATOR
Monitor Solana network gas fees as market-wide health indicator. High gas = busy network = good. Low gas = quiet market = lower confidence.
